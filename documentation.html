1. Introduction & Concepts

What is the Compliance Workflow Manager V7?

The Compliance Workflow Manager V7 is a powerful tool for creating, managing, and tracking compliance workflows. Version 7 introduces the revolutionary dynamic template system, allowing you to define custom workflow structures with unlimited flexibility.

Key Features

  • Dynamic Templates: Create workflows with any number of levels and custom properties
  • 4 Creation Methods: From Template, Empty, Copy, or Linked workflows
  • Dual Modes: Creation mode for building, Execution mode for tracking
  • Template Snapshots: Each workflow preserves its structure independently
  • Linked Workflows: Synchronize structure across multiple workflows
  • Rich Features: Tags, attachments, grading, progress tracking, and more

The Template-Based Revolution NEW IN V7

Version 7 transforms how you work with workflows by introducing Workflow Templates. Instead of a rigid 3-level structure (Rules â†’ Actions â†’ Evidences), you now define your own structure:

Version 6 (Old) Version 7 (New)
Fixed 3-level structure Unlimited custom levels (1-10+)
Same features for all workflows Custom features per template
Hard to reuse structures Reusable templates
Manual consistency Template ensures consistency

Real-World Example

Scenario: Your organization needs to track NIST CSF compliance across departments.

Old Way (V6): Create multiple workflows, manually ensure they all have the same structure.

New Way (V7): Create one "NIST CSF Compliance" template, then create workflows for each department from that template. All workflows share the same structure, and you can link them to synchronize changes!

Understanding Template Snapshots

When you create a workflow from a template, the system creates a snapshot (copy) of that template and stores it with the workflow. This means:

Important: Changing a template does NOT affect existing workflows created from it. Each workflow has its own independent structure stored as a snapshot. This protects your workflows from accidental changes.

  • Each workflow is independent and protected
  • Template changes don't affect existing workflows
  • Workflows can have different structures even from the same template
  • You can always see what template a workflow was based on

3. Workflow Creation Methods

Version 7 offers four powerful methods to create workflows, each suited for different scenarios:

Method 1: From Template RECOMMENDED

Create a workflow based on a predefined template structure.

When to Use

  • You have a custom template that fits your needs
  • You want a specific multi-level structure
  • You need consistency across multiple workflows
  • You're working with industry standards (NIST, ISO, etc.)
  • Click "New" button
  • Select "From Template" (default option)
  • Enter a workflow name
  • Select a template from the dropdown
  • Click "Create Workflow"

Example Use Case

Scenario: Your organization has a "NIST CSF Compliance" template with 5 levels: Functions â†’ Categories â†’ Subcategories â†’ Controls â†’ Evidence.

Action: Create workflows named "Finance Department NIST", "IT Department NIST", etc., all from the same template.

Result: All departments have identical structures, making compliance tracking consistent.

Method 2: Empty Workflow (Quick Start)

Create a simple 1-level workflow instantly, without needing a template.

When to Use

  • You need a quick checklist or simple tracking
  • You're just exploring the system
  • You don't have templates set up yet
  • You need a flat list without hierarchy
  • Click "New" button
  • Select "Empty Workflow (Quick Start)"
  • Enter a workflow name
  • Click "Create Workflow"

The system automatically creates a simple template with all properties enabled. Your level will be called "Item" (singular) / "Items" (plural).

Method 3: Copy Existing Workflow

Duplicate an existing workflow with all its structure and units.

When to Use

  • You want to reuse a workflow as a starting point
  • You need multiple similar workflows (e.g., per department)
  • You want to preserve the exact structure and content
  • You need a new version of an existing workflow
  • Click "New" button
  • Select "Copy Existing Workflow"
  • Enter a name for the copy
  • Select the source workflow to copy from
  • Click "Create Workflow"

Important: Copying creates a complete independent duplicate. The copy includes:

  • The template snapshot (structure)
  • All units with their names and properties
  • All attachments (links, images, notes)
  • New unique IDs for everything

The copy is NOT linked to the original - changes to one won't affect the other.

Method 4: Linked Workflow (Synchronized) V7 FEATURE

Create a workflow that synchronizes its structure with other workflows.

When to Use

  • Multiple workflows should share the same structure
  • You need synchronized updates across workflows
  • You're tracking the same requirements across departments/teams
  • Changes in one should reflect in all linked workflows
  • Click "New" button
  • Select "Linked Workflow (Synchronized)"
  • Enter a name for the new workflow
  • Select the workflow to link to
  • Click "Create Workflow"

How Linking Works: When you add, remove, or rename units in Creation mode and save, those changes automatically propagate to all linked workflows. However, execution state (checkmarks, grades) remains independent for each workflow.

Linked Workflow Example

Scenario: You're tracking GDPR compliance across 5 regional offices.

Setup:
1. Create "GDPR Compliance - HQ" from template
2. Create "GDPR Compliance - Europe" as linked workflow
3. Create "GDPR Compliance - Asia" as linked workflow
4. All three are now synchronized

Result: When HQ adds a new control, it appears in all regional workflows automatically. But each region tracks their own completion status independently.

Linked vs Copy: Use Copy when you want independent workflows. Use Linked when you want synchronized structure but independent tracking.

4. Understanding Template-Based Workflows

What is a Template Snapshot?

Every workflow in V7 contains a template snapshot - a complete copy of the template it was created from. This snapshot defines:

  • Number of levels: How many hierarchical levels the workflow has
  • Level names: What each level is called (singular and plural)
  • Level properties: What features are available at each level
  • Workflow properties: What workflow-level features are enabled

Snapshot Protection

Template snapshots protect your workflows:

  • Editing a template doesn't change existing workflows
  • Deleting a template doesn't affect workflows created from it
  • Each workflow is self-contained and independent
  • You can always see what template a workflow was based on

6. Creation vs Execution Modes

Understanding the Two Modes

The workflow manager operates in two distinct modes, each with a specific purpose:

Mode Purpose What You Can Do
Creation Build & modify structure Add/edit/delete units, change names, modify structure
Execution Track progress Check off items, enter grades, view progress

Creation Mode

Purpose: Define and organize the workflow structure.

In Creation Mode You Can:

  • Add new units at any level
  • Edit unit names and descriptions
  • Change icons and unit IDs
  • Delete units and restructure
  • Add tags and attachments
  • Set up grading structure
  • Create/rename/delete workflows

In Creation Mode You Cannot:

  • Check off "Done" checkboxes
  • Mark items as complete
  • Track execution progress

Execution Mode

Purpose: Track completion and progress.

In Execution Mode You Can:

  • Check off "Done" checkboxes
  • Enter/update grades
  • View progress bars
  • See cumulative grades
  • View completion statistics
  • Filter by tags
  • Export to boards

In Execution Mode You Cannot:

  • Add or delete units
  • Change structure
  • Edit names or descriptions
  • Modify the workflow layout

Switching Between Modes

The mode toggle is located in the top-right corner of the screen:

  • Look for the toggle switch labeled "Creation | Execution"
  • Click the switch or the mode label to toggle
  • The interface updates immediately
  • Creation-only buttons disappear in Execution mode
  • Done checkboxes become clickable in Execution mode

Important: Always save your work before switching modes! Changes in one mode don't automatically save to the other.

Visual Differences

Element Creation Mode Execution Mode
Add Buttons Visible Hidden
Delete Buttons Visible Hidden
Done Checkboxes Disabled (visual only) Clickable
Name Fields Editable Read-only
Progress Bars Shows estimate Shows actual progress
Management Buttons Visible Hidden

Workflow for Using Both Modes

Typical Workflow

  • Creation Mode: Build the workflow structure, add all units, organize hierarchy
  • Click "Save Structure"
  • Switch to Execution Mode
  • Start checking off completed items

  • If you need to add more items, switch back to Creation Mode
  • Add the new items, save structure
  • Switch to Execution Mode to continue tracking

9. Linked Workflows V7 FEATURE

What Are Linked Workflows?

Linked workflows are workflows that share the same structural changes. When you modify the structure of one linked workflow (add/delete/rename units), those changes automatically propagate to all other linked workflows in the group.

What Gets Synchronized:

  • Adding new units (structure)
  • Deleting units (structure)
  • Renaming units (names only)
  • Changing unit descriptions
  • Unit IDs and icons

What Stays Independent:

  • Done checkboxes (execution state)
  • Grades
  • Tags
  • Attachments (links, images, notes, comments)
  • Workflow-level properties

Creating Linked Workflows

There are two ways to create linked workflows:

Method 1: Link to Existing Workflow

  • Click "New" button
  • Select "Linked Workflow (Synchronized)"
  • Enter a name for the new workflow
  • Select an existing workflow to link to
  • Click "Create Workflow"

Method 2: Link Multiple Workflows

  • Create first workflow: "Team A - GDPR"
  • Create linked: "Team B - GDPR" (linked to Team A)
  • Create linked: "Team C - GDPR" (linked to Team A)
  • All three are now in the same link group

Identifying Linked Workflows

When a workflow is linked, you'll see:

  • "Linked" indicator badge next to the workflow selector
  • "Unlink" button becomes visible in the toolbar
  • Both indicators appear in Creation mode only

How Synchronization Works

Synchronization happens automatically when you save changes in Creation mode:

  • Open any linked workflow
  • Switch to Creation mode
  • Make structural changes (add, delete, rename units)
  • Click "Save Structure"
  • System automatically propagates changes to all linked workflows
  • All linked workflows now have the same structure

Template Protection: The system only synchronizes workflows that share the same template. You cannot accidentally sync a 3-level workflow with a 5-level workflow - they must be based on the same template structure.

Execution State Preservation

One of the most powerful features of linked workflows is smart execution state preservation:

Smart Synchronization Example

Scenario: You have 3 linked workflows. Team A has completed 10 items.

Action: You add a new unit "Security Assessment" to Team A's workflow.

Result:

  • All 3 workflows get the new "Security Assessment" unit
  • Team A's 10 completed items remain checked
  • Team B and Team C's completion states are preserved
  • The new unit is unchecked for all teams

Intelligent Matching: The system uses smart ID matching to preserve execution states even when structure changes. Your team's progress is safe!

Unlinking Workflows

To remove a workflow from a link group:

  • Select the workflow you want to unlink
  • Make sure you're in Creation mode
  • Click the "Unlink" button in the toolbar
  • The workflow is removed from the link group
  • "Linked" indicator disappears
  • Workflow continues to function independently

Important: Unlinking is permanent for that workflow. It won't receive future structural changes from the group. However, the workflow itself is not deleted - it just becomes independent.

12. Frequently Asked Questions

General Questions

Q: What's the difference between V6 and V7?

A: Version 7 introduces dynamic templates, allowing unlimited custom workflow structures. V6 had a fixed 3-level structure (Rules â†’ Actions â†’ Evidences). V7 lets you define 1-10+ levels with custom names and properties.

Q: Do I need to create templates before using workflows?

A: No! You can use "Empty Workflow (Quick Start)" which automatically creates a simple template for you. However, creating templates first gives you more control over structure and features.

Q: Can I change a workflow's template after creation?

A: No. Each workflow has a template snapshot that's fixed at creation time. To use a different template, create a new workflow. However, you can modify the structure manually in Creation mode.

Template Questions

Q: If I edit a template, do existing workflows change?

A: No! Each workflow has its own independent template snapshot. Template changes only affect new workflows created from that template.

Q: Can I have different properties at different levels?

A: Yes! This is a key feature. For example, you might have:

  • Level 1: Icons, progress bars (no Done checkbox)
  • Level 2: Icons, grades (no Done checkbox)
  • Level 3: Everything (Done checkbox, attachments, etc.)

Linked Workflow Questions

Q: What's the difference between Copy and Linked workflows?

A:

  • Copy: Creates independent duplicate. Changes in one don't affect the other.
  • Linked: Creates synchronized workflow. Structural changes in one update all linked workflows.

Q: Can I link workflows created from different templates?

A: No. Workflows can only be linked if they share the same template. This prevents structural mismatches.

Q: If I unlink a workflow, what happens to my data?

A: Nothing! The workflow continues to function normally with all its data intact. It just stops receiving structural updates from the link group.

Q: Can I re-link an unlinked workflow?

A: Not directly. Once unlinked, you would need to create a new linked workflow from an existing one in the link group.

Mode Questions

Q: Can I edit unit names in Execution mode?

A: No. Execution mode is for tracking progress only. Switch to Creation mode to edit structure, names, or properties.

Q: Why can't I see the Add/Delete buttons?

A: You're probably in Execution mode. Switch to Creation mode using the toggle in the top-right corner.

Q: Do I need to save separately in each mode?

A: Yes! "Save Structure" (Creation) and "Save Execution" (Execution) are separate. Changes in one mode don't automatically save data for the other.

Data Questions

Q: Where is my workflow data stored?

A: Data is stored in JSON files in the data/ folder:

  • workflows.json - Workflow structures
  • executions.json - Completion states
  • templates.json - Templates
  • workflow-links.json - Link relationships

Q: How do I backup my workflows?

A: Copy the entire data/ folder to a backup location. These JSON files contain all your workflow, template, and execution data.

Q: Can multiple users work on the same workflow?

A: The system saves to files, so concurrent editing can cause conflicts. For multi-user scenarios:

  • Use linked workflows (one per user/team)
  • Coordinate edit times to avoid conflicts
  • Consider implementing a backend database in production

Performance Questions

Q: How many units can a workflow have?

A: Technically unlimited, but for best performance:

  • Keep workflows under 500 units
  • Use multiple workflows for large projects
  • Consider using tag filtering for large workflows

Q: The system seems slow. What can I do?

A: Try these steps:

  • Clear browser cache
  • Reduce workflow size (split into multiple workflows)
  • Minimize image sizes in attachments
  • Close other browser tabs
  • Update to the latest browser version

13. Troubleshooting

Common Issues

"Create Workflow" button not responding

Possible Causes:

  • Name field is empty
  • No template selected (in Template mode)
  • No source workflow selected (in Copy/Linked mode)

Solution: Fill in all required fields marked with * before clicking "Create Workflow".

Changes not saving

Possible Causes:

  • Didn't click "Save Structure" or "Save Execution"
  • Browser closed before save completed
  • File permission issues on server

Solution: Always click the Save button and wait for confirmation. Check browser console for errors.

Can't see Add/Delete buttons

Cause: You're in Execution mode

Solution: Switch to Creation mode using the toggle in the top-right corner.

Linked workflows not synchronizing

Possible Causes:

  • Workflows have different templates
  • Changes made but not saved
  • Workflows not actually linked (check for "Linked" indicator)

Solution: Verify workflows are in the same link group, share the same template, and save after making changes.

"Template not found" error

Cause: Template was deleted but workflow still references it

Solution: Workflows contain template snapshots and should still work. If you see this error, check browser console for details.

Getting Help

Debug Mode: Open browser Developer Tools (F12) and check the Console tab for detailed error messages. These can help identify issues quickly.

Data Recovery: If workflows become corrupted:

  • Check the data/ folder for JSON files
  • Restore from backup if available
  • Edit JSON files manually if needed (advanced users)
